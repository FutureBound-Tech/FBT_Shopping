================================================================================
                    AI BRAIN - DETAILED STUDY & REPLICATION GUIDE
================================================================================
Application: E-Commerce Product Catalog (Fashion/Textile)
Purpose: AI-powered image analysis, product description generation, 
         image variation generation, and background removal.

================================================================================
1. ARCHITECTURE OVERVIEW
================================================================================

The AI brain consists of 3 major subsystems:

  A) IMAGE ANALYSIS ENGINE     - Reads product images and generates metadata
  B) IMAGE GENERATION ENGINE   - Creates product image variations
  C) BACKGROUND REMOVAL ENGINE - Removes backgrounds via URL transformation

All AI features are exposed via REST API endpoints under /api/ai/* and consumed
by a React frontend admin panel (ProductEdit.jsx).

3. FEATURE 1 - IMAGE ANALYSIS ENGINE (AI Image Reader)
================================================================================

PURPOSE:
  Given a product image URL, the AI analyzes the image in detail and generates:
  - A catchy professional product title
  - A detailed premium product description (fabric, style, occasions)
  - Relevant tags (color, category, style, material)
  - Product category classification (sarees or dress-materials)

ENDPOINT:
  POST /api/ai/analyze
  Headers: Authorization: Bearer <JWT_TOKEN>
  Body: { "image_url": "https://..." }

IMPLEMENTATION (backend/controllers/aiController.js):

  Step 1: Validate that image_url is provided in request body.

  Step 2: Call Groq chat completions API with multimodal input:
    - System prompt: "You are a professional fashion e-commerce catalog expert.
                      You must provide product analysis in valid JSON format only."
    - User message: Contains TWO parts:
      Part A (text): Prompt asking to analyze the clothing/textile product image
                     and return JSON with keys: title, description, tags, category
      Part B (image_url): The product image URL for visual analysis
    - Model: meta-llama/llama-4-scout-17b-16e-instruct
    - Temperature: 0.5 (balanced creativity/accuracy)
    - Max tokens: 1024
    - Response format: json_object (enforces JSON output)

  Step 3: Parse the JSON response and return to frontend.

  Step 4: Frontend (ProductEdit.jsx) populates form fields with AI results:
    - name <- data.title
    - description <- data.description
    - category <- data.category
    - tags <- data.tags

FRONTEND TRIGGER:
  Button: "Magic AI Fill" (Sparkles icon)
  Only visible when an image_url is set.
  Shows loading spinner "AI Reading..." during analysis.

KEY CODE SNIPPET (Backend):
  const chatCompletion = await groq.chat.completions.create({
    messages: [
      {
        role: 'system',
        content: 'You are a professional fashion e-commerce catalog expert. You must provide product analysis in valid JSON format only.'
      },
      {
        role: 'user',
        content: [
          {
            type: 'text',
            text: `Analyze this clothing/textile product image and provide:
            1. A catchy professional product title.
            2. A detailed, premium product description highlighting possible fabric, style, and occasions.
            3. A list of relevant tags (color, category, style, material).
            
            Return the response ONLY as a JSON object with the keys: "title", "description", "tags" (array of strings), and "category" (either "sarees" or "dress-materials").`
          },
          {
            type: 'image_url',
            image_url: { url: image_url }
          }
        ]
      }
    ],
    model: 'meta-llama/llama-4-scout-17b-16e-instruct',
    temperature: 0.5,
    max_tokens: 1024,
    response_format: { type: 'json_object' }
  });

  const result = JSON.parse(chatCompletion.choices[0].message.content);

4. FEATURE 2 - IMAGE GENERATION ENGINE (Magic Images)
================================================================================

PURPOSE:
  Given a base product image, generate 3 professional product photo variations
  using AI image generation with different styles/angles.

ENDPOINT:
  POST /api/ai/generate-variations
  Headers: Authorization: Bearer <JWT_TOKEN>
  Body: { "image_url": "https://..." }

IMPLEMENTATION (backend/controllers/aiController.js):

  Step 1: Validate base image_url is provided.

  Step 2: DESCRIBE THE BASE IMAGE (Groq Vision):
    - Send the image to Groq with a concise description prompt
    - Prompt: "Analyze this product image and give a highly detailed visual 
              description (colors, texture, clothing type, patterns, lighting). 
              Keep it under 60 words. Return ONLY the description text."
    - Model: meta-llama/llama-4-scout-17b-16e-instruct
    - This description is used to maintain visual consistency in generated images

  Step 3: BUILD 3 VARIATION PROMPTS using the base description:
    Variation 1: "Ultra-realistic professional studio product lifestyle 
                  photography of [DESCRIPTION], featured in a premium boutique 
                  setting, cinematic lighting, 8k resolution, elegant, 
                  side angle view"
    Variation 2: "Macro close-up photography of the fabric texture and 
                  intricate patterns of [DESCRIPTION], highlighting the fine 
                  details and weave, studio lighting, 8k high-fidelity"
    Variation 3: "Beautifully draped view of [DESCRIPTION] displayed on an 
                  invisible mannequin, professional ecommerce catalog style, 
                  soft white background, perfectly lit, alternate look"

  Step 4: GENERATE IMAGE URLs via Pollinations.ai:
    - Format: https://image.pollinations.ai/prompt/{ENCODED_PROMPT}?width=1024&height=1024&seed={RANDOM}&nologo=true&enhance=true
    - Each variation gets a random seed for unique results
    - Pollinations.ai generates images on-demand from text prompts (free, no API key needed)

  Step 5: Return array of 3 generated image URLs to frontend.

FRONTEND FLOW (ProductEdit.jsx):
  - Button: "Magic Images" (Wand2 icon, golden gradient)
  - Opens "AI Magic Studio" modal with loading animation
  - Shows progress bar, loading states, feature badges (4K Rendering, Professional Angles, Fabric Analysis)
  - Displays 3 generated images in a grid
  - Each image has two actions:
    * "Regenerate" - re-generates that specific slot
    * "Keep" - adds the image to the product's gallery

KEY CODE SNIPPET (Backend):
  // 1. Get description
  const descriptionResponse = await groq.chat.completions.create({
    messages: [
      { role: 'system', content: 'Describe product images for AI image generation...' },
      { role: 'user', content: [
        { type: 'text', text: 'Analyze this product image and give a highly detailed visual description...' },
        { type: 'image_url', image_url: { url: image_url } }
      ]}
    ],
    model: 'meta-llama/llama-4-scout-17b-16e-instruct',
  });

  const baseDescription = descriptionResponse.choices[0].message.content;

  // 2. Build prompts and generate URLs
  const variationPrompts = [
    `Ultra-realistic professional studio product lifestyle photography of ${baseDescription}...`,
    `Macro close-up photography of the fabric texture...`,
    `Beautifully draped view of ${baseDescription}...`,
  ];

  const generatedImages = variationPrompts.map((p, i) => {
    const seed = Math.floor(Math.random() * 1000000);
    return `https://image.pollinations.ai/prompt/${encodeURIComponent(p)}?width=1024&height=1024&seed=${seed}&nologo=true&enhance=true`;
  });

5. FEATURE 3 - BACKGROUND REMOVAL ENGINE
================================================================================

PURPOSE:
  Remove the background from product images using Cloudinary's built-in 
  AI background removal transformation.

IMPLEMENTATION (ProductEdit.jsx - Frontend Only):

  This feature uses Cloudinary's URL-based transformation, NOT a custom AI model.

  Step 1: Check if the image URL is a Cloudinary URL (contains "cloudinary.com")
    - If not Cloudinary, show error: "Background removal only works for Cloudinary images"

  Step 2: Toggle background removal by manipulating the Cloudinary URL:
    - ADD background removal:
      Original: https://res.cloudinary.com/CLOUD_NAME/image/upload/v123/folder/image.jpg
      Modified: https://res.cloudinary.com/CLOUD_NAME/image/upload/e_background_removal/v123/folder/image.jpg
      (Insert "e_background_removal/" after "/upload/")

    - REMOVE background removal:
      Remove "/e_background_removal/" from the URL

  Step 3: The modified URL is set as the new image_url, Cloudinary processes 
          the transformation server-side on-the-fly.

FRONTEND UI:
  - Toggle switch labeled "Remove Background AI"
  - Only appears when image is a Cloudinary URL
  - Switches between original and background-removed versions instantly
  - No backend endpoint needed - pure URL transformation

KEY CODE SNIPPET (Frontend):
  const toggleBackgroundRemoval = () => {
    if (!image_url) return;
    
    const isCloudinary = image_url.includes('cloudinary.com');
    if (!isCloudinary) {
      setError('Background removal only works for Cloudinary images');
      return;
    }

    if (!removeBg) {
      const parts = image_url.split('/upload/');
      const newUrl = `${parts[0]}/upload/e_background_removal/${parts[1]}`;
      setImageUrl(newUrl);
    } else {
      const newUrl = image_url.replace('/e_background_removal/', '/');
      setImageUrl(newUrl);
    }
    setRemoveBg(!removeBg);
  };

CLOUDINARY REQUIREMENTS:
  - Cloudinary account with "Background Removal" add-on enabled
  - Add-on uses Cloudinary's AI (likely based on U2-Net or similar segmentation models)
  - May require paid plan depending on usage volume

11. AI MODEL DETAILS
================================================================================

Model Used: meta-llama/llama-4-scout-17b-16e-instruct
Provider: Groq (https://groq.com)
Type: Multimodal (text + image input)
Parameters: 17 billion
Context: 16k tokens
Capabilities:
  - Vision: Can analyze product images, identify colors, patterns, fabric types
  - Text generation: Produces structured JSON output
  - Fashion domain: Understands clothing categories, textile terminology

Why Groq?
  - Extremely fast inference (sub-second responses for image analysis)
  - Free tier available with generous limits
  - Native multimodal support (image + text in single request)
  - JSON mode for structured output

Alternative Models That Would Work:
  - meta-llama/llama-4-maverick-17b-128e-instruct (larger context)
  - llama-3.2-11b-vision-preview (Meta's vision model)
  - GPT-4o (OpenAI, paid)
  - Claude 3.5 Sonnet (Anthropic, paid)
